# 山海终端模块检查设计

## 目标

为山海终极终端新增“模块检查”设置。开启后，玩家手持终端左键多方块控制器时，只显示该结构中可挂载机器模块的位置，并使用青色边框标记。

## 行为

- 设置保存到终端自身 NBT，默认关闭。
- 模块检查只影响左键结构扫描，不改变右键搭建、拆解、AE 请求或功能部件替换。
- 开启后，左键扫描过滤普通缺失方块、阻塞方块、替换项和仓室提示。
- 所有模块挂载位均显示，包括空位、占位机壳位置和已经安装模块的位置。
- 模块挂载位统一使用 `0x00FFFF` 青色边框，并向玩家报告识别到的位置数量。
- 未识别到模块挂载位时发送明确提示，并清除此前的结构高亮，避免旧标记残留。

## 识别方式

从结构谓词收集每个位置的候选方块。候选方块对应的机器只要实现 GTLCore 的 `IModularMachineModule`，该结构位即为模块挂载位。

识别结果按方块缓存，避免每次左键扫描重复实例化机器。该方式不依赖注册名中的 `module` 字样，也无需维护山海模块 ID 列表，因此可兼容后续新增模块和其他实现同一接口的模块。

## 改动范围

- `ShanhaiUltimateTerminalConfig`：新增模块检查 NBT 读写。
- `ShanhaiUltimateTerminalBehavior`：新增设置开关与模块检查报告分支。
- 终端模块识别辅助类：集中维护候选方块到模块类型的判定缓存。
- 中英文语言文件：新增设置名称和扫描提示。
- 终端专项测试：覆盖开关持久化、接口识别、左键过滤及青色标记。

## 验证

1. 运行终端专项测试。
2. 运行完整 Gradle 构建。
3. 实机开启模块检查，左键原始终焉引擎，确认 16 个模块位均显示青色框。
4. 关闭模块检查后，确认左键恢复原有结构缺失及仓室分类显示。
