# EAEP 配方类型搜索推荐设计

## 目标

EAEP 样板上传界面的搜索框除主机名称外，也能匹配星律样板总成暴露的配方类型名称，并优先推荐对应主机。

## 已确认行为

1. 搜索不采用严格过滤，始终保留 EAEP 原本可选择的主机。
2. 当前搜索词非空时，同时匹配主机名称、配方类型 ID、配方类型路径和 `recipe_type_names.json` 映射名称。
3. 配方类型命中的主机即使被 EAEP 原名称过滤排除，也要补回当前结果列表。
4. 配方类型命中的主机排在非命中主机之前；同组内部保持 EAEP 原有顺序。
5. 右侧类型提示优先显示当前搜索命中的配方类型，其后显示其他类型，最多显示 5 个。
6. 清空搜索框后，恢复按本次上传样板的配方类型排序和提示，不丢失原推荐语义。

## 数据流

1. 服务端继续复用现有 EAEP provider 列表扩展包，下发每个星律主机暴露的配方类型 ID。
2. 客户端保留按供应器显示名称合并后的配方类型映射。
3. EAEP 执行原 `applyFilter` 后，山海逻辑遍历完整分组列表，将当前搜索词命中的配方类型主机补入过滤结果。
4. 补入完成后对四个并行列表同步排序：provider ID、名称、总槽位、同名主机数量。
5. 页面按钮和右侧提示继续读取排序后的同一索引，保证点击目标不发生错位。

## 边界条件

- 搜索词为空：不补入额外结果，使用上传样板类型作为推荐依据。
- 搜索词没有任何配方类型命中：保持 EAEP 原过滤或回退结果。
- 主机名称与配方类型同时命中：结果只保留一份。
- 多个同名主机：沿用 EAEP 分组结果，不拆散同名主机计数。
- 映射文件不存在或读取失败：仍可通过完整 ID 和路径匹配。

## 验证

1. 回归测试锁定当前搜索词优先于上传样板类型，清空搜索后恢复上传类型。
2. 回归测试锁定类型匹配主机会补入结果且四个并行列表保持对齐。
3. 回归测试锁定不实现严格过滤，非匹配主机仍可选择。
4. 完整构建后部署九周目，实机搜索映射中文名、类型路径和无匹配文本。
