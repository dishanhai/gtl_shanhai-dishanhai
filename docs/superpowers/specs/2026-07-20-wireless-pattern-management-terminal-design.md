# 山海无线样板管理终端设计

## 目标

新增独立物品 `gt_shanhai:wireless_pattern_management_terminal`，提供 ExtendedAE 扩展样板管理终端的核心管理能力，并为星律样板总成增加远程配置入口。

终端必须同时支持：

- 作为独立无线终端右键打开。
- 安装进 `ae2wtlib:wireless_universal_terminal` 后作为一个终端模块打开。
- 放入与无线通用终端相同规则的 Curios 槽位。
- 通过独立快捷键从手持、无线通用终端、Curios 或玩家背包中查找并打开。
- 完全继承 AE2WTLib 的网络绑定、耗能、距离、权限和量子桥卡规则。

## 非目标

- 不修改 ExtendedAE 原有扩展样板管理终端的行为。
- 不通过 Mixin 向原终端追加按钮或点击逻辑。
- 不建立第二套 AE 网络绑定、供能或权限系统。
- 不允许客户端通过任意坐标直接打开机器界面。
- 第一版不向普通 AE2、ExtendedAE 或第三方样板总成写入专用配置。

## 总体架构

采用独立终端实现，复用 AE2 和 AE2WTLib 的公开终端基础设施，不依赖 ExtendedAE 客户端界面的私有字段。

主要组件：

1. `ShanhaiWirelessPatternManagementTerminalItem`
   - 独立物品入口。
   - 使用 AE2WTLib 无线终端链接、能量和范围能力。
   - 可注册为无线通用终端模块。

2. `ShanhaiPatternManagementMenuHost`
   - 独立物品和无线通用终端共用的菜单主机。
   - 向 `PatternAccessTermMenu` 提供当前 AE Grid。

3. `ShanhaiPatternManagementMenu`
   - 继承 AE2 `PatternAccessTermMenu` 的样板容器同步和库存操作。
   - 维护 `serverId -> 服务端样板容器` 的权威映射。
   - 向客户端同步容器位置、维度、朝向和是否为星律。
   - 处理远程配置请求并执行完整服务端校验。

4. `ShanhaiPatternManagementScreen`
   - 原生 Minecraft/AE2 界面，不使用 HTML。
   - 提供样板总成分组、输入搜索、输出搜索、样板增删和位置高亮。
   - 对星律容器提供槽位中键和库存拉取按钮。

5. `ShanhaiPatternRemoteConfigPacket`
   - 客户端只发送当前菜单中的 `serverId`、操作类型和可选槽号。
   - 服务端通过当前菜单追踪器重新解析实际容器，不信任客户端坐标。

6. `ShanhaiPatternRemoteUIFactory`
   - 为星律构造两个远程 LDLib 页面：单槽催化剂页和库存拉取页。
   - 页面直接绑定目标星律现有数据处理器，不复制库存或配置状态。

## 终端界面

### 基础管理

界面保留以下能力：

- 按样板总成分组显示样板。
- 输入物品/流体搜索。
- 输出物品/流体搜索。
- 普通点击、Shift 点击和创造模式复制等 AE2 样板管理行为。
- 样板总成位置高亮和坐标提示。
- 终端样式及样板总成显示设置。

### 星律槽位中键

玩家在星律对应的样板槽上按鼠标中键时：

1. 客户端读取 `PatternSlot` 的容器 `serverId` 和槽号。
2. 发送 `OPEN_SLOT_CATALYST` 请求。
3. 服务端确认该容器仍属于当前菜单可见的 AE Grid。
4. 服务端确认目标是星律或能解析到星律本体的星律代理。
5. 服务端校验槽号有效后打开该槽的物品催化剂、流体催化剂和实际配方数量配置页。

中键由山海终端优先消费，不触发创造模式样板复制。普通样板总成的中键不发送远程请求。

### 库存拉取按钮

每个样板总成标题行预留远程操作区域。目标为星律时显示库存图标按钮：

- 点击发送 `OPEN_STOCK_INPUT` 请求。
- 打开目标星律现有的 32 格库存拉取设置和高级拉取间隔配置。
- 普通样板总成不显示该按钮。
- 星律代理显示按钮，但服务端操作实际绑定的星律本体。

### 返回管理终端

两个远程配置页都提供返回按钮。返回时重新通过原终端物品或无线通用终端模块打开管理终端；若终端已移除、掉落、失去链接或能量不足，则关闭页面并提示原因。

## 无线终端集成

### 独立打开

独立物品沿用 AE2WTLib 无线终端主机：

- 必须已链接到安全终端。
- 按 AE2WTLib 规则消耗能量。
- 受无线范围、无限范围卡和量子桥卡约束。
- 使用 AE2 安全权限检查。

### 无线通用终端

通过 `WUTHandler.addTerminal` 注册独立终端模块：

- 模块 ID 使用 `shanhai_pattern_management`。
- 无线通用终端内显示“山海无线样板管理终端”。
- 安装后复用同一个菜单主机、菜单和界面。
- 不在无线通用终端 NBT 中保存第二份 AE 链接。

### Curios 与快捷键

物品使用与无线通用终端相同的 Curios 可装备规则。

独立快捷键触发后按以下优先级查找可打开入口：

1. 主手和副手中的独立终端。
2. 玩家背包中的无线通用终端，且已安装山海终端模块。
3. Curios 中的独立终端或已安装模块的无线通用终端。
4. 玩家背包中的独立终端。

找到第一个可用入口后，由服务端执行 AE2WTLib 的链接、权限、距离和能量校验。校验失败时继续尝试下一个候选；全部失败时只显示最高优先级失败原因。

## 服务端安全与一致性

远程操作必须满足：

- 请求来自当前打开的山海样板管理菜单。
- `serverId` 存在于该菜单当前的容器追踪表。
- 容器仍连接到玩家当前访问的 AE Grid。
- 玩家拥有对应 AE 安全权限。
- 目标维度已加载，目标方块实体或部件仍存在。
- 目标为 `RecipeTypePatternBufferPartMachine`，或可解析到它的星律代理。
- 槽号在目标星律当前有效样板槽范围内。

任一条件失败都拒绝请求，不修改机器状态。配置页面打开后若目标卸载或断网，页面进入只读失效状态并自动关闭。

## 星律接口调整

星律新增窄接口，不向终端暴露内部数组：

- `createRemoteSlotCatalystUI(Player, int slot, RemoteReturnTarget)`
- `createRemoteStockInputUI(Player, RemoteReturnTarget)`
- `isValidRemotePatternSlot(int slot)`
- `resolveRemotePatternOwner()`

现有本地侧栏和远程页面复用同一组 Widget 构造方法，避免本地 UI 与远程 UI 逻辑分叉。

## 兼容性

- AE2WTLib 是必需运行依赖；ExtendedAE 通过 AE2 公共样板容器接口兼容，不作为山海终端的实现依赖。
- 不修改 ExtendedAE 类，不注册面向其私有字段的 accessor/mixin。
- ExtendedAE 升级只影响显示语义参考，不影响山海终端的数据追踪基础。
- 普通 AE2/ExtendedAE 样板总成仍可在终端中管理样板，但没有星律专用远程按钮。

## 测试

### 单元与源码测试

- 终端物品、菜单、菜单主机和界面注册完整。
- WUT 模块 ID 和菜单类型唯一。
- 中键请求携带正确的 `serverId + slot`。
- 库存按钮只为星律容器生成。
- 服务端不接受客户端坐标。
- 普通容器、失效 serverId、错误槽号和跨 Grid 请求均被拒绝。
- 快捷键查找顺序固定，候选失败后能继续回退。
- 正式源码不依赖 ExtendedAE 客户端私有字段。

### 实机验收

1. 独立终端链接、充能后右键打开。
2. 将模块安装进无线通用终端，可以从通用终端列表打开。
3. 独立终端或通用终端放入 Curios 后，快捷键可以打开。
4. 无线范围外、无能量、未链接和无权限时按 AE2WTLib 原规则拒绝。
5. 星律样板槽中键打开正确槽号的催化剂页，修改后只影响该槽。
6. 星律标题行库存按钮打开对应星律的库存拉取页。
7. 星律代理入口解析到真实星律本体。
8. 普通样板总成不显示星律按钮，中键不影响样板。
9. 远程页面返回按钮能回到同一个管理终端入口。
10. 多维度、量子桥卡、终端掉落、目标卸载和断网场景不会越权或崩溃。

## 完成标准

- 四种打开入口均可用：独立右键、无线通用终端、Curios、快捷键。
- 管理终端基础样板操作与 ExtendedAE 使用体验等价。
- 星律槽位中键和库存拉取按钮都能修改真实远端星律。
- 所有远程请求由服务端根据当前 AE 菜单追踪器重新验证。
- 构建、定向测试和九周目实机验收通过。
