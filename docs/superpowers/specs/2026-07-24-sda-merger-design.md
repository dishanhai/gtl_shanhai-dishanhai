# SDA 合并器设计

## 目标

新增一个原生 AE2 网络设备 `gt_shanhai:sda_merger`。设备接收两个超级磁盘阵列（SDA），自动生成一个全新的 SDA；输出包含两个输入的全部主存储条目和全部 `virtual_cells`，且不修改输入 SDA 的任何 UUID 或后端数据。

## 设备形态

- 设备使用 AE2 方块、方块实体和菜单体系，不注册为 GTCEu 机器。
- 方块实体继承 `AENetworkInvBlockEntity`，主节点设置 `GridFlags.REQUIRE_CHANNEL`。
- 设备占用 1 个频道，空闲功耗为 `1 AE/t`。
- 节点在线时才允许合并；断网、缺电或无频道时保留槽位内容并停止工作。
- GUI 包含两个 SDA 输入槽和一个只读输出槽，不提供手动开始按钮。
- 侧面只允许插入输入槽，底面只允许抽取输出槽。
- 破坏设备时必须掉落三个槽位中的全部物品。

## 存储语义

### 主存储

两个输入 SDA 的主存储后端均以只读快照读取。所有物品键和流体键统一按 `AEKey` 合并：

```text
output[key] = left[key] + right[key]
```

数量运算全程使用 `BigInteger`，不得经由 `long`、`KeyCounter` 或 AE2 常规 `insert` 分批搬运，避免截断超出 `Long.MAX_VALUE` 的数量。

输出 SDA 使用新的随机顶层 UUID，并设置为已经认领的运行时 SDA。输入 SDA 的顶层 UUID、NBT、SavedData 和 ContentStore 均不得改写。

### virtual_cells

两个输入 SDA 的 `virtual_cells` 列表按原顺序拼接到输出：左侧全部单元在前，右侧全部单元在后。不同虚拟单元不互相折叠，因为其 `type`、`bytes` 和容量边界属于单元自身语义。

每个输出虚拟单元执行深拷贝：

1. 保留原单元的 `type` 和 `bytes`。
2. 读取原 `data_uuid` 对应的完整存储内容；旧式内联 `keys/amts` 同样需要读取。
3. 为输出单元生成新的随机 `data_uuid`。
4. 将内容写入新 UUID 对应的 SavedData。
5. 输出 NBT 只保存轻量 `data_uuid`、`type` 和 `bytes`，不继续携带重型内联内容。

原虚拟单元的 UUID 和后端保持不变。虚拟单元内容复制也使用 `BigInteger`，不能沿用当前 `VirtualCellStorage` 中会降为 `long` 的保存路径。

## 合并事务

合并逻辑集中在无 UI 依赖的 `SdaMergeService`，方块实体只负责触发与提交。一次合并按以下顺序执行：

1. 验证网络在线、两个输入均为数量 1 的 SDA、输出槽为空。
2. 验证两个输入顶层 UUID 不相同；相同则拒绝，防止同一后端被重复计数。
3. 验证所有来源 `virtual_cells` 的已存在 `data_uuid` 不重复；重复则拒绝，防止共享子后端被复制多次。
4. 完整读取两个输入快照并在内存中计算主存储结果及虚拟单元复制计划。
5. 预生成输出顶层 UUID 和所有输出虚拟单元 UUID。
6. 将新虚拟单元和新顶层 SDA 内容写入 SavedData。
7. 构造输出 ItemStack，并再次确认输入槽引用及 UUID 未变化、输出槽仍为空。
8. 先放入输出槽，再清空两个输入槽，最后标记方块实体保存。

若步骤 1 至 7 任一步失败，不消耗输入。为步骤 6 新增的 SavedData 记录提供删除接口；提交前失败时清理本次新 UUID，避免孤儿后端。输入槽只在输出已经成功放入后清空。

## 调度与性能

- 使用 AE2 `IGridTickable`，无可合并任务时进入休眠。
- 输入槽变化或主节点恢复在线时唤醒设备。
- 每次唤醒最多处理一对 SDA，不轮询 SavedData 全表。
- 合并只遍历两个输入实际拥有的键和虚拟单元，复杂度为 `O(left entries + right entries)`。
- 输出槽未取走前保持休眠，不重复创建后端或 UUID。

## UI 与状态

- 界面布局为“输入 SDA + 输入 SDA → 输出 SDA”。
- 输入槽只接受 `SuperDiskArrayItem`，输出槽禁止放入。
- 状态显示包括：等待输入、网络离线、输出槽占用、UUID 冲突、正在合并、合并完成、合并失败。
- 方块活动状态由主节点在线状态驱动，使用 AE2 风格金属外壳和独立正面材质。
- 方块名、状态和 tooltip 提供中英文语言键。

## 注册与资源

- 在 `DShanhaiAE2Blocks` 注册方块、方块物品和方块实体。
- 在现有菜单注册入口注册 `SdaMergerMenu`，客户端通过 AE2 `InitScreens` 注册 `SdaMergerScreen`。
- 加入山海创造模式物品栏。
- 提供方块状态、方块模型、物品模型、掉落表、屏幕样式和正面材质。
- 本次不猜测整合包平衡材料，不内置合成配方；注册 ID 可直接供 KubeJS/数据包添加配方。

## 边界行为

- 空 SDA 可以参与合并。
- 主存储重复键按 `BigInteger` 相加；物品和流体都支持。
- `virtual_cells` 不扁平化进主存储，也不按类型合并成一个单元。
- 任一输入不是 SDA、堆叠数量不是 1、输出非空或网络离线时均不处理。
- 相同顶层 UUID 或重复子 `data_uuid` 均拒绝处理并保留输入。
- 设备重启、区块卸载和 GUI 关闭不会清空输入或输出。
- 自动化不能从输入槽抽取正在等待处理的 SDA，也不能向输出槽插入物品。

## 验证标准

1. 设备未连接网络、缺电或无频道时不合并，恢复在线后自动继续。
2. 两个输入主存储中的物品和流体均进入输出，重复键数量精确相加。
3. 超过 `Long.MAX_VALUE` 的主存储数量合并后保持精确。
4. 两边全部 `virtual_cells` 出现在输出，顺序稳定，类型、容量和内容不变。
5. 输出顶层 UUID 及所有输出子 `data_uuid` 均为新随机值，且不等于任一来源 UUID。
6. 合并前后两个输入的 UUID 和后端快照完全一致。
7. 同顶层 UUID 或重复子 UUID 的输入被拒绝，不产生输出，也不消耗输入。
8. 输出槽占用、保存失败或读取异常时不消耗输入，不留下本次新建的 SavedData 记录。
9. 漏斗/管道只能从侧面输入 SDA，并只能从底面抽取输出。
10. 破坏设备时三个槽位中的物品均正常掉落。
11. 专项测试、全量测试和 `clean build --no-daemon` 通过，部署 Jar 与构建产物哈希一致。
