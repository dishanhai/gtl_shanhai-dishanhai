# 终焉枢纽专项

* <ItemLink id="gt_shanhai:maintenance_hatch" />

<ItemImage id="gt_shanhai:maintenance_hatch" scale="4" />

## 说明

<ItemLink id="gt_shanhai:maintenance_hatch" /> 是这套分类里最特殊的一类，所以单独放，不并入普通输出或仓室。

它不是单纯的维护仓，也不是普通物流节点，而是把维护、耗时、并行、跨配方线程、电压绕过、环境条件、研究需求、概率、温度和产出倍率这些能力统合到一起的终局节点。你可以把它理解成“把很多分散职责收束到一个中心位”的枢纽。

它还能作为主机上的能力插槽使用：放入物质模块后，枢纽会根据模块决定可调耗时范围和最大并行上限；再放入 <ItemLink id="dishanhai:time_reversal_protocol" />、<ItemLink id="gtladditions:astral_array" />、<ItemLink id="gtladditions:compressed_astral_array" />、<ItemLink id="dishanhai:big_tear" />、<ItemLink id="dishanhai:create_mk" /> 相关的解锁物，就会逐步解锁线程、概率、产出倍率这些更高阶的能力。

换句话说，这套系统不是一个单纯的维护口，而是一整套终局控制台。玩家先装模块，再按需要开关绕过项，最后用界面微调速度、并行、线程和输出倍率。

这种东西适合放在已经成体系的终局多方块里，作为功能汇聚点来用。它的说明不该和一般输出口混写，否则玩家会误以为它只是一个高级接口。

## 具体功能

<Column gap="2" fullWidth={true}>

### 耗时倍率

> 枢纽可以把机器的基础耗时拉快或拉慢，范围由当前插入的物质模块决定。\
> 低级模块给较小的可调区间，高级模块给更宽的区间，玩家能直接在界面里用按钮调节。

</Column>

<Column gap="2" fullWidth={true}>

### 并行数量

> 枢纽自带可调并行上限。\
> 模块决定并行的天花板，玩家可以在上限内手动调节当前并行值。\
> 对很多终局产线来说，这比单纯堆机器更直接。

</Column>

<Column gap="2" fullWidth={true}>

### 跨配方线程

> 枢纽还能接管跨配方线程。\
> 装入 <ItemLink id="gtladditions:astral_array" />、<ItemLink id="gtladditions:compressed_astral_array" /> 或 <ItemLink id="dishanhai:time_reversal_protocol" /> 后，线程会提升；如果放入 <ItemLink id="dishanhai:create_mk" />，还会把线程和并行一起推到极限。\
> 这个能力是它和普通维护仓最不同的地方之一。

</Column>

<Column gap="2" fullWidth={true}>

### 电压与条件绕过

> 枢纽可以按开关绕过电压、环境条件、研究需求、温度和概率限制。\
> 这个功能不是所有玩法都要开，但当你已经进入终局结构调试时，它能把很多本来会卡住机器的条件直接收束掉。

</Column>

<Column gap="2" fullWidth={true}>

### 产出倍率

> 当同时插入 <ItemLink id="dishanhai:create_mk" /> 和 <ItemLink id="dishanhai:big_tear" /> 时，产出倍率会解锁。\
> 解锁后可以继续调倍率，机器产物会被整体放大。\
> 这不是普通维护仓能做的事，它更像终局版的控制中枢。

</Column>

<Column gap="2" fullWidth={true}>

### 数据访问

> 枢纽还带数据访问能力，意味着它能参与更高阶的结构判定和配方可用性检查。\
> 对一些需要“这台机器已经被提升到终局层级”的系统来说，这个能力很关键。

</Column>

## 使用方式

<Column gap="2" fullWidth={true}>

枢纽理论支持所有结构 因为其注册了所有可能的仓室映射(输入，输出，并行，天球...)，再往它的插槽中放入合适的模块、<ItemLink id="gtladditions:astral_array" />、<ItemLink id="gtladditions:compressed_astral_array" />、<ItemLink id="dishanhai:time_reversal_protocol" /> 或创造类解锁物。之后再根据目标玩法调整界面里的开关和数值。

</Column>

<Column gap="2" fullWidth={true}>

> 如果你只放了枢纽本体，它仍然能作为维护与基础控制节点工作\
> 但想要高并行、线程、产出倍率和概率绕过这些完整能力，就得把对应解锁物补齐。

</Column>

## 物质模块系统详解

### 模块的基本作用

物质模块是枢纽最核心的配置部件。插入不同等级的物质模块会解锁并行与线程能力。等级越高，提供的并行数越多。

**关键规则**：
* 机器本身的并行值高于枢纽提供时，两者会叠加计算
* 最大并行不能超过 `Long.MAX_VALUE`（9223372036854775807），否则会发生数值溢出

| 5 | 嬗变物质模块 | 2,048 |

## 特殊机器安装功能

### 安装至伪神之锻炉

* 锁定 114514 小时运行时间
* 移除枢纽后会削减
* 解锁模块限制
* 模块不再需要伪神锻运行

### 安装至(伪神缎附属模块)恒星聚变异化器

* 永不锁定物质异化配方

### 安装至鸿蒙之眼/奥术星图

* 氢氦消耗到一定限度后不再消耗

### 安装至奥术星图（gtladditions 3.1c+ 新功能）

* 压缩星阵产出概率强制提升至 100%
* 不需要放入 42,000 个星阵
* 直接开启星阵压缩配方


### 安装至子模块机器（需开启配置文件）——注意目前无需枢纽只需开启对应配置即可

* 允许子模块脱离主机独立运作
* 适用对象：太空电梯子模块、超时空子模块、卫星工厂、伪神模块、终焉引擎模块
* 模块改为全局行为

## 配方产出倍率与能源配置

### 产出倍率

* **范围**：1.0 ～ 5.0 倍（需创造模块 + 大反冲解锁）
* **叠加效果**：输出 1 物品 → 增幅 5 倍得 5 物品，与引力波叠加可达 15 倍

### 能源电压等级

* **可配置范围**：ULV ～ MAX+16（使用 `long` 类型）

### 性能提示

* **警告**：功能 Mixin 过多会导致主机成型缓慢或出现异常，需要重新放置主机以恢复正常

### 终焉模式（配置文件可开启）

在终焉模式下，线程会转换为产出倍率：

* 最高 4G 线程（创始现实修改模块）对应最高产出倍率 4G
* 允许进一步叠加，最高倍率计算：
  * 基础：4G 线程（创始现实修改模块）
  * 伪神倍率：×15
  * 枢纽倍率：×5
  * 引力波倍率：×3
* **总计最高产出倍率：18,000 亿倍**

## 线程增强系统

### 星阵槽（线程基数）

星阵提供线程的基础数值：

| 星阵类型 | 数量 | 线程输出 | 倍率 |
|---------|------|--------|------|
| 普通星阵 | 1 格 | 1,046,272 线程 | ×1 |
| 压缩星阵 | 1 格 | 1,607,073,792 线程 | ×1,536 |
| 世线信标 | 1 格 | Long.MAX_VALUE 线程 | 全线程解锁 |

### 线程增强槽（倍率乘法）

世线残片可以进一步增强线程输出：

| 增强件 | 数量 | 倍率 |
|-------|------|------|
| 世线残片 Lv1 | 64 个 | ×64 倍率 |
| 世线残片 Lv2 | 64 个 | ×256 倍率 |
| 世线残片 Lv7 | 64 个 | ×262,144 倍率 |
| 寰宇并行超限器 | - | 并行/线程强制 MAX |

### 物质模块槽（耗时 + 并行）

17 级物质模块（入门至创始）提供：
* 耗时倍率：0.001 ～ 0.50
* 并行上限：256 ～ 无限（Long.MAX_VALUE）

### 线程计算公式

```
最终线程 = 星阵基数 × 残片倍率
上限钳制：Long.MAX_VALUE
```

### 计算示例

**示例 1**：普通星阵 ×1 + 世线残片 Lv1 ×64
```
1,046,272 × 64 = 66,961,408 线程
```

**示例 2**：压缩星阵 ×1 + 世线残片 Lv7 ×64
```
1,607,073,792 × 262,144 = 已触及上限（Long.MAX_VALUE）
```
