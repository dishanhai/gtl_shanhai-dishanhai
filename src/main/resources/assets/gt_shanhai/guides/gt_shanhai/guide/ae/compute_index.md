# 算力仓

* <ItemLink id="gt_shanhai:logical_compute_hatch" />

<BlockImage id="gt_shanhai:logical_compute_hatch" scale="4" />

## 说明

<ItemLink id="gt_shanhai:logical_compute_hatch" /> 是专门用来解决“算力输入不足”这类痛点的内置算力接口。

它不是普通输入仓，也不是物流口，而是直接把算力接进机器控制器内部的专用部件。接入后，<ItemLink id="gt_shanhai:logical_compute_hatch" /> 会稳定提供无限算力，让配方继续运行，不再被外置算力输入偶发的判定问题卡住。

如果你用过 GTCEu 或 GTMThings 的算力输入仓室，见过明明已经补了算力却还是提示“算力不足”，那它就是为这种情况准备的。想要的是稳定、直接、不会莫名掉链子的算力供给，就用 <ItemLink id="gt_shanhai:logical_compute_hatch" />；想要物项转运、缓存、请求补货，就别把它当成仓室来理解。

## 适合场景

* 需要给多方块结构提供稳定算力时。
* 需要避开外置算力输入可能出现的“算力不足”误判时。
* 需要一个只做算力、不掺物流职责的内置接口时。
